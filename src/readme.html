Readme

Store name:

Israel's Soccer Team's Store

What are you selling?

Products for Israel's Soccer Team's fans and tickets for the team's games.

What additional page(s) did you add? How to operate it?

Tickets - Buying tickets to the team's games, done by filtering month\year to find a game, then selecting a stand in the stadium with available tickets, then selecting a ticket, adding it ti the ticket cart, and "paying" for it in checkout for tickets.
Because of its' workload of creating, it was considered by you (Ohad) as more then one screen.

News - shows selected news about the national team from news-api.

Contact us - allows the visitor to send us emails.
You (Ohad) can look at the contact router code for the details and see the mails at https://ethereal.email/.

What was hard to do?

For me, the hardest thing to do was handeling tickets' frontend logic - it is a "wizard" with 4 stages,
that need to communicate with 3 different servers' logics and handle weird edge cases.
Next in line was the tickets' frontend visuals. Maybe (or even probably) it's because of my of css skills, but some parts of it took a lot of time to finish (stand selection).
Last in this line is the tests - they demended deep understanding of the backend's logic, which i didn't have when i started writing them.

Specify all the different routes your app supports

  • Authentication Routes:
    • POST /api/register - Register a new user
    • POST /api/login - User login
    • POST /api/logout - User logout
  • User Data Routes:
    • GET /api/user-data - Retrieve authenticated user data
  • Game Routes:
    • GET /api/games - Retrieve all games
  • Ticket Routes:
    • GET /api/tickets - Retrieve tickets (with optional gameId, stand, ticketId filters)
    • POST /api/tickets/purchase - Purchase tickets
  • Ticket Cart Routes:
    • GET /api/ticket-cart/view - View ticket cart
    • POST /api/ticket-cart/add-to-cart - Add ticket to cart
    • POST /api/ticket-cart/remove - Remove ticket from cart
  • Store Routes:
    • GET /api/store/products - Retrieve all products
    • GET /api/store/search - Search products
  • Cart Routes:
    • GET /api/cart/view - View cart
    • POST /api/cart/add-to-cart - Add item to cart
    • POST /api/cart/removeAll - Remove all items from cart
    • POST /api/cart/complete-purchase - Complete purchase
  • News Route:
    • GET /api/news - Retrieve news articles
  • Contact Route:
    • POST /api/contact - Submit contact form
  • Admin Routes:
    • GET /api/admin/activities - Retrieve admin activities
    • POST /api/admin/add-product - Add a new product
    • POST /api/admin/remove-product - Remove a product
    • GET /api/admin/products - Retrieve all products
    • POST /api/admin/remove-game - Removes a game
    • POST /api/admin/add-game - Add a new game

Installation & Setup

Prerequisites:

  • Node.js
  • Sqlite3

Step 1: Install Dependencies

  • npm install in the /fullstack_final_project folder
  • npm install in the /fullstack_final_project/tests folder

Step 2: Running Test

  • In /fullstack_final_project folder, run the server using:
    npm start server.js
  • Important: Tests rely on initiated products, so please run tests before using rest of the store's options (which may change the products).
  • In /fullstack_final_project folder, use:
    node tests/test.js

Step 3: Using the application:

  • The app should now be running on http://localhost:3000.
  • Notes:
    • You have to login in order to buy.
    • Games available for buying tickets are set to 08/24 and 09/24 (so you can find them easily on tickets.html).
